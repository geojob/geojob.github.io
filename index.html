Mechanical Standing Wheelchair

Designed and tested a full Mechanical standing wheelchair that utilizes linear springs to achieve three different standing angles. The stress/strain analysis was completed on Ansys and the chair was modeled on Solidoworks/CATIA.

Using the dials on the double acting cylinders, the speed of the overall system transition was controlled. The plunger was attached to the armrests on the torso frame of the wheelchair to allow the user to move from a standing to a seated position at will.

Evolutionary Algorithm trained Walking Bot

An 8 vertex bot was trained using an evolutionary/genetic algorithm to essentially "walk" as far as possible.

Vertices were added to the bot based on a random coefficient to increase diversity within the population. A population of random individuals was created and the best walking iterations were reproduced with one another and mutated based on the mutation coefficient.

This was achieved by learning the spring coefficients between mass points (vertices). The mutation allowed the bot to travel towards the peak whereas the crossover within the reproduction method allowed to jump peaks to find the global maximum.

Robot Manipulation

ROS was used in conjuction with Python and C++ to create an environment to test Cartesian Control and inverse kinematics on a robot arm. The 7 dof Kuka LWR arm was used to test the algorithms.

The algorithm also ensured to check for self-inflicting collisions. The module uses ROS publisher and subscriber nodes for geometry messages used to calculate the transforms and Jacobians. An optional secondary objective was added to the command properties to control the end effector velocity. This was achieved using null space control.

As for the inverse kinematics algorithm, a timed iterative process was used to achieve the desired end effector orientation starting from a random set of joint values. In the event that the timer ran out, a new random set of joint values was used to start the process again until a set number of attempts are completed.

Instance Segmentation for Recycling

A U-Net structured CNN was used to generate masks given RGB images of recyclable containers on a conveyor belt. The data used included 200 RGB images along with annotated masks. These were then trained on the U-net model consisting of several 2D convolution layers along with dropout and batch normalization to reduce the possibility of overfitting.

A testing accuracy of 98.2% and a validation accuracy of 90.4% was achieved with this particularly small dataset. However, the mean intersection over union was only around 60% which is due to the lack of training data as well having to downsize the images from 1920x1080 to 128x128 to achieve faster compute times. A simple OpenCV script was then used to identify the objects with bounding boxes and calculate an approciamte rotation angle.

Snake Team at Dr. Ciocarlie's ROAM Lab

I was the team lead for the snake team research project, primarily focusing on controls and firmware, at the ROAM (Robotic Manipulation and Mobility) lab under Professor Ciocarlie's supervision.

This is a modular platform that uses multiple series elastic actuators that are linked together to achieve a sinusoidal/serpenoid motion. Each link is controlled by a Teensy microcontroller that recieves data from multiple sensors including a magnetic encoder, and IMU and an optical sensor. The sensor readings along with their corresponding noises are used in a Kalman Filter along with constraints like the fixed distance between sensors of consequent links to determine the state of the system at any given time.

The link controllers are all connected via a CAN networks and communicate with a central controller that is connected to a ROS workstation over Serial communication. The ROS workstation is used to log the feedback from the sensors as well as send step commands/sinusoidal wave commands to each of the links. These commands are processed by the PID control algorithm on each of the link controllers to reach the goal position.

Some of the challenges being faced at the moment include the overcrowded CAN network due to the high frequency at which commands and feedback is being sent. I am currently optimizing the interrupt architecture to improve the performance and obtain a smooth tracking within the allowed bandwidth for the SEA springs.

Once this is done, the MPC algorithm described in this paper as well as an RL algorithm, being concurrently developed and tested/simulated in MuJoCo will be compared with respect to path planning capabilities.

Motion Planning and RRT

Similar to the previous ROS project, the module uses a 7 dof KUKA lwr arm to test motion planning algorithms in avoiding obstacles.

A grid based map for the configuration space was used to determine the ideal "path" of joint values to get to the desired cartesian coordinates. The obstacles are encoded into this configuration space.

Instead of using Djikstra's algorithm or A*, a stochastic algorithm [RRT] was used due to the high dimensionality of the configuration space that corresponds to a 7 dof arm. Once the ideal tree is found, the trajectory is then shortened by moving from node to node checking for collisions and deleting redundant nodes in between. Then the joint values corresponding to the nodes of the tree are published via ROS to the arm to be visualized in ROS Gazebo.

Self Balancing Cartpole using Deep Q Learning

For this project, a Deep Q-learning network was used to balance a cartpole.

The model was made using PyTorch using three fully connected layers with the first two using ReLU activation funtions. 2D Robot State Estimation

This project uses an Extended Kalman Filter to estimate the state of a 2D robot [Localization]. The state consists of the robot's position and orientation in a given time window. The dyanamic model is defined by updating the x, y and theta values for every time step.

Since the problem is using a landmark based feature map as well as being 2 dimentsional, an extended kalman filter was more practical, in the interest of computational resources, than for example a particle filter. Using an initial arbitrary state and covariance matrix for the state of the robot, sensor data was read from the ROS topic. This data along with the landmark coordinates and the given noise covariances were used to fill in the jacobian matrices for both the predicted state estimate and the covariance estimate. Then the innovations and Kalman gain were computed based on each of the sensor readings being received. Finally, the state and the covariance were updated so that the state estimating robot can be updated to be as close to the real robot being tracked.

It was really interesting to see that the noise model used affects the overall localization in even a simple problem such as this one. Working on a real-life system, like the snake project, has shown the importance of denoising the sensors as well as modelling the noise appropriately to obtain a farily accurate estimate of the robot's state.

The next obvious step would be to stack a mapping component for SLAM applications. Another intriguing problem is dealing with non-holonomic kinematic systems without the use of any dynamics.

Forward Dynamics of a 3 Link Arm

The goal of this project was to accurately learn the forward dynamics of a three-link arm using deep learning.

The arm in red has been dynamically modeled to represent the motion of a free hanging three-link arm. The arm in blue has no constraints or dynamic modelling applied except for the deep learning model that attempts to predict where it's supposed to be to closely match the arm in red. A separate script publishes a constant torque to the three joints of both these arms.

First data was collected as the state of arm 3xjoint velocities, 3xjoint angles in relation to the 3xcommanded joint torques. So, the 6 dimensional state was superpositioned with the 3 dimensional command published at that state as the input. The resulting 6 dimensional state was recorded as the output.

The model was made using PyTorch using three fully connected layers activated by the ReLU function. Adam was used as the optimizer along with MeanSquaredError as the loss function. Adaptive learning rates were not applied to the dataset being fairly uniform and complete.

Since the dataset was collected for the first 10 seconds of commanded torques, the model is able to properly predict the state of the blue arm for around the same time as it tends to fail for time steps after it. Although the dataset was collected for a descrete set of torque values, it was interesting to see that model was still able to predict the state of the blue arm for an unseen torque value as long as it is within the range of values trained on.

Stochastic Latent Actor Critic for Stacking

This project was my Robot Learning capstone project with Professor Shuran Song at Columbia University.

The goal of the project was to successfully train the agent (Sawyer robot arm) using raw image data to stack geometrically complex objects. Deepmind's stacking environment was used along with Mujoco as the setup for this project.

In order to use image data in such a large environment, a stochastic latent space was used to obtain more meaningful information extraction and noise filtering.

An actor critic model was used in conjunction with the latent variable model to train the agent on the task of stacking. This was chosen instead of a more popular/simpler policy gradient method to increase sample efficiency, maintain a low variance as well as a good level of entropy/exploration.

Reward engineering played a huge role especially since sparse rewards were used. The rewards were separated into distinct "stages" to appropriately guide the agent to learn the task at hand.

The SLAC method was compared against the MPO-based agent used in Deepmind's paper where the former converged in far fewer timesteps at a higher overall return. This approach effectively demonstrates that using a latent variable model greatly increases the performance of an agent especially when learning long-horizon tasks using raw image data.

Conveyor Belt Live Object Tracking

This is one of the few tasks I completed during my intership at a smart-recycling startup, Evtek. A Mask-RCNN model was used to predict the labels for each of the recycled containers seen by the camera. The predictions were then used in a Kalman Filter as priors to predict the next cartesian position of the same container in the following frame.

The state matrix is low dimensional since only the position and velocities of the objects were being tracked along the x and y axis. The system noise covariance matrix was set to 0 since the conveyor belt is set to a constant speed and stationed in a controlled environment (recycling facility). The Kalman Filter was then tuned by tweaking the measurement noise covariance and inital system covariance matrices.

With respect to the colors, if an object is identified by the dashed box as a single color over multiple frames, then the KF tracker is successfully identifying and tracking the object. The solid red box is scaled model output bounding box used simply as a sanity check.

This method allows for an accurate and consistent method to track, count and identify the different recycled objects. The tracking module additionally allows to improve the object prediction model by checking for irregular changes in the number of objects between two consecutive frames.
